Understanding Web Frameworks

By Franco Andaya

A web framework (WF) or web application framework (WAF) is a software framework designed to support the development of web applications, including web services, web resources, and web APIs.

Web frameworks provide a standard way to build and deploy web applications on the World Wide Web. They aim to automate the overhead associated with common activities performed in web development, such as database access, templating, and session management.

Types of Web Frameworks

Web frameworks can be broadly categorized into frontend and backend frameworks.

Frontend Frameworks

Frontend frameworks are used to create the user interface of the website that users interact with. They are typically written in JavaScript and are responsible for the functionality and interactivity of the website.

Some popular frontend frameworks include:

  • Angular: An open-source TypeScript-based framework created and maintained by Google. It is used for developing websites and Progressive Web Apps (PWAs).
  • Vue.js: An open-source JavaScript-based framework that supports Model-View-ViewModel (MVVM) architecture. It is easy to learn and has large community support.
  • React: Built by Facebook, React organizes frontend code into components, making it reusable and maintainable.

Backend Frameworks

Backend frameworks are used on the server side of the application. They handle requests, databases, and communication with APIs.

Some popular backend frameworks include:

  • Django: An open-source Python-based framework that provides a speedy development of web applications. It is based on Model Template View (MTV) architecture.
  • Ruby on Rails: An open-source Ruby-based framework that follows the Model View Controller (MVC) architecture. It is known for its scalability and security
  • Spring Boot: An open-source Java-based framework that enables developers to develop production-ready and standalone applications
